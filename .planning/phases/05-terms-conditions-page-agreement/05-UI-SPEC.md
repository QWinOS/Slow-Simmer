# Phase 5 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | `b2jI` — radix-nova style, neutral base, large radius |
| Component library | radix-ui (v1.6.1 monorepo) |
| Icon library | lucide |
| Font (heading) | Bodoni Moda (`--font-heading`, weights 400–700) |
| Font (body) | Jost (`--font-sans`, weights 300–600) |

**Source:** `components.json`, `app/layout.tsx`, `npx shadcn info`

Note: The preset's default Geist font is overridden by explicit Bodoni Moda + Jost declarations in `app/layout.tsx`. The `/terms` page must use `font-heading` and `font-sans` Tailwind classes — NOT Playfair Display SC or Karla (those were aspirational brand references in the requirements, not the built design).

## Spacing Scale

All values are multiples of 4, using TailwindCSS default scale:

| Token | Value | Usage |
|-------|-------|-------|
| `xs` | 4px | Inline padding within checkbox label, separator margins |
| `sm` | 8px | Card inner padding, clause section gap |
| `md` | 16px | Default element spacing, paragraph margins |
| `lg` | 24px | Section padding, card padding (`p-6`) |
| `xl` | 32px | Layout gaps between major sections |
| `2xl` | 48px | Page-top/bottom spacing |
| `3xl` | 64px | Page-level section breaks |

**Exceptions:** The established `min-h-[44px]` / `min-w-[44px]` touch-target size from NavBar and Footer icon buttons is an accessibility standard already in the codebase. It is NOT re-implemented in this phase (no new icon-only controls). The checkbox itself uses the existing `size-4` (16×16px) with the Radix touch-target expansion via `after:absolute after:-inset-x-3 after:-inset-y-2` already baked into `components/ui/checkbox.tsx`.

**Source:** TailwindCSS defaults + existing `components/NavBar.tsx`, `components/Footer.tsx`, `components/ui/checkbox.tsx`

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | `text-sm` (14px) / `text-base` (16px) | 400 (regular) | 1.5 | Jost (`font-sans`) |
| Label / Checkbox | `text-sm` (14px) | 400–500 | 1.5 | Jost (`font-sans`) |
| Subheading (clause title) | `text-base` / `text-lg` (16–18px) | 600 (semibold) | 1.3 | Bodoni Moda (`font-heading`) |
| Page heading (T&C title) | `text-2xl` / `text-3xl` (24–30px) | 700 (bold) | 1.2 | Bodoni Moda (`font-heading`) |

**Details:**

- **Body** — T&C clause content, "Last updated" string, "Back to Registration" link: `font-sans text-sm sm:text-base leading-relaxed text-muted-foreground`
- **Clause headings** (h2 per clause): `font-heading text-base sm:text-lg font-semibold`
- **Page heading** (h1): `font-heading text-2xl sm:text-3xl font-bold text-foreground`
- **Checkbox + label** — Use `text-sm leading-relaxed text-muted-foreground` for the agreement label. The linked "Terms & Conditions" text uses the same size with `underline underline-offset-4 hover:text-primary` styling.
- **"Back to Registration"** link: `text-sm sm:text-base font-medium` styled as a primary-colored anchor or inline button.
- **"Last updated"** line: `text-xs sm:text-sm text-muted-foreground`

**Source:** Existing patterns in `app/page.tsx` (hero heading: `font-heading text-4xl`), `components/RegistrationForm.tsx` (section titles: `font-heading text-lg font-bold`, body: `text-muted-foreground`), and `app/layout.tsx` (font declarations).

## Interaction States

| Element | States | Behavior |
|---------|--------|----------|
| T&C checkbox | unchecked (default), checked, focus-visible, invalid (aria-invalid) | Radix checkbox with `onCheckedChange` coerced to boolean. Focus ring via `focus-visible:border-ring focus-visible:ring-3 focus-visible:ring-ring/50`. Invalid state via `aria-invalid:border-destructive`. |
| "Terms & Conditions" link | default, hover, visited, focus-visible | Opens `/terms` in `target="_blank" rel="noopener noreferrer"`. Underlined with `underline-offset-4`. Hover transitions to `text-primary`. |
| "Back to Registration" link | default, hover, focus-visible | Navigates to `/#form` (same tab). Hover shows underline or color transition. |
| NavBar "Terms" item | active (on `/terms` route), inactive (on `/` route) | Active state uses the same underline indicator as other sections (`after:scale-x-100` + `text-foreground`). Determined by `usePathname()`, not IntersectionObserver. |
| Checkbox error state | visible on form submit attempt when unchecked | Error message rendered by existing `FieldError` component below the checkbox. Also shown in the `ErrorSummary` alert at the top of the form. |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Checkbox label | `<label htmlFor={field.name}>` wrapping the text, linked to checkbox via `id` |
| Link opens new tab | `target="_blank" rel="noopener noreferrer"` — prevents tab-napping |
| Focus indicators | Native browser focus ring + Radix focus-visible styles on checkbox |
| Color contrast | All text meets WCAG AA against card background. Error text uses `text-destructive` which passes contrast ratios defined in `globals.css`. |
| Reduced motion | No new animations in this phase. T&C page is static content. |
| Keyboard navigation | Tab order: checkbox → link inside label → remaining form fields → submit. The `/terms` page link opens in a new tab — user returns to the form via tab/window switch. |
