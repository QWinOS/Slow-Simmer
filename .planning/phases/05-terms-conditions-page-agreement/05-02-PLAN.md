       ```ts
       const NAV_SECTIONS = [
         { id: "about", label: "The Club" },
         { id: "gallery", label: "Gallery" },
         { id: "videos", label: "Film" },
         { id: "membership", label: "Join Us" },

       ] as const
       ```

    3. **Add `usePathname()` call** inside the component body, after the `useState` hooks and before `useEffect`:
       ```ts
       const pathname = usePathname()
       const isTermsPage = pathname === "/terms"
       ```

    4. **Guard the IntersectionObserver `useEffect`** — add early return at the top of the effect body:
       ```ts
       useEffect(() => {
         if (isTermsPage) return // no section elements on /terms
         const sectionIds = ["hero", ...NAV_SECTIONS.map((s) => s.id), "form"]
         // ... rest of existing effect unchanged
       }, [isTermsPage])
       ```
       Add `isTermsPage` to the dependency array: `}, [isTermsPage])`.

    5. **Override active section on `/terms`** — after the `useState` hooks, compute the effective active section:
       ```ts
       const effectiveActiveSection = isTermsPage ? "terms" : activeSection
       ```
       Then replace all references to `activeSection` in the JSX with `effectiveActiveSection`. Specifically:
       - In the desktop nav links: change `activeSection === section.id` to `effectiveActiveSection === section.id`
       - In the mobile menu: change `activeSection === section.id` to `effectiveActiveSection === section.id`

       Do NOT change the `setActiveSection` calls in the IntersectionObserver callback — those still set the `activeSection` state for the main page.

    6. **"Terms" anchor link** — the `href` for the "Terms" nav item should navigate to `/terms` (not `#terms` since `/terms` is a full route, not a hash anchor):
       Since `NAV_SECTIONS` is a `const` array and the render loop maps over it generically using `section.id` to build `href="#{section.id}"`, the "terms" entry needs special handling. The simplest approach: change the href construction in both desktop and mobile render loops to use `/terms` when `section.id === "terms"`, else `#${section.id}`:
       ```tsx
       href={section.id === "terms" ? "/terms" : `#${section.id}`}
       ```
       This appears in two places: the desktop `<a>` (line ~78) and the mobile `<a>` (line ~124).
