    ### Aadhar Input (per D-12, D-13, D-14)

    Custom masked input with eye toggle:

    ```typescript
    const [showAadhar, setShowAadhar] = useState(false)
    ```

    Format function (store raw digits, display formatted):
    ```typescript
    function formatAadhar(value: string): string {
      const digits = value.replace(/\D/g, "").slice(0, 12)
      if (digits.length === 0) return ""
      return digits.replace(/(\d{4})(?=\d)/g, "$1 ")
    }
    ```

    The Controller's render function uses InputGroup:
    ```typescript
    <Field data-invalid={fieldState.invalid}>
      <FieldLabel htmlFor="aadhar">
        Aadhar Number <span className="text-destructive">*</span>
      </FieldLabel>
      <InputGroup>
        <InputGroupInput
          id="aadhar"
          type={showAadhar ? "text" : "password"}
          value={formatAadhar(field.value || "")}
          onChange={(e) => {
            const raw = e.target.value.replace(/\D/g, "").slice(0, 12)
            field.onChange(raw)
          }}
          onBlur={field.onBlur}
          name={field.name}
          ref={field.ref}
          aria-invalid={fieldState.invalid}
          autoComplete="off"
        />
        <InputGroupAddon align="inline-end">
          <InputGroupButton
            size="icon-xs"
            variant="ghost"
            onClick={() => setShowAadhar(!showAadhar)}
            aria-label={showAadhar ? "Hide Aadhar" : "Show Aadhar"}
            type="button"
          >
            {showAadhar ? <EyeOffIcon className="size-4" /> : <EyeIcon className="size-4" />}
          </InputGroupButton>
        </InputGroupAddon>
      </InputGroup>
      <FieldDescription className="text-xs text-muted-foreground">
        Your Aadhar is used only for event check-in
      </FieldDescription>
      {fieldState.invalid && <FieldError errors={[fieldState.error]} />}
    </Field>
    ```
    Note: `EyeOffIcon` and `EyeIcon` from lucide-react — import as `Eye, EyeOff` and rename in usage or use directly.

    Per UI-SPEC: Eye toggle button is `type="button"` to prevent form submission. Input type toggles between "password" and "text". aria-label is dynamic.

    ## ErrorSummary Banner (per D-09, D-19)

    ```typescript
    function ErrorSummary() {
      const errors = form.formState.errors
      const errorEntries = Object.entries(errors)
      if (errorEntries.length === 0) return null

      return (
        <div role="alert" className="rounded-lg border border-destructive/50 bg-destructive/10 p-4 mb-6">
          <p className="font-medium text-destructive mb-2">
            Please fix the following errors:
          </p>
          <ul className="list-disc pl-5 text-sm text-destructive space-y-1">
            {errorEntries.map(([field, error]) => (
              <li key={field}>
                {(error as { message?: string })?.message ?? "Invalid value"}
              </li>
            ))}
          </ul>
        </div>
      )
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Form input fields → Zod validation | Untrusted user input crosses into validation layer |
| Form component → React Context | Validated but sensitive data (Aadhar) enters shared context |
| toast.success() | Ephemeral UI feedback — no data in toast |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-07 | Tampering | Form submitted without Zod validation | medium | mitigate | `form.handleSubmit()` only fires `onSubmit` after Zod schema passes — browser's native validation is secondary, Zod is primary gate. `mode: "onBlur"` gives early feedback but submit is always gated. |
| T-02-08 | Information Disclosure | Aadhar shown when eye toggle is in "reveal" state | low | accept | Eye toggle requires intentional user action to reveal. Default state is masked (type="password"). Shoulder-surfing risk is inherent to any reveal mechanism and is standard UX (same as password fields). |
| T-02-09 | Spoofing | XSS via paste into About/Social fields | low | mitigate | React's JSX escaping in `{data}` prevents script injection. Social field is URL-validated via Zod. About is text-only with 200 char limit. Error messages are Zod-sourced (not user input reflected back unsafely). |
| T-02-10 | Repudiation | User pastes non-digit characters into Aadhar or Contact | low | mitigate | onChange handler strips non-digits with `.replace(/\D/g, "")`. Zod schema enforces full-digit constraint. Combined defense-in-depth. |
</threat_model>
