### Error Summary Banner

| State | Visual |
|-------|--------|
| Hidden | `display: none` — not rendered when `formState.errors` is empty |
| Visible | `role="alert"`, `rounded-lg border border-destructive/50 bg-destructive/10 p-4`, positioned at top of form above first section |
| Content | Heading "Please fix the following errors:" + unordered list of error messages |
| Focus behavior | After submit attempt with errors, `form.handleSubmit` auto-focuses first invalid field via `setFocus()`. Summary banner is NOT focused — it's a `role="alert"` announcement for screen readers. |

### Auto-Scroll (Success)

1. User clicks Submit → validation passes → `form.handleSubmit(onSubmit)` fires
2. `setRegistrationData(data)` stores in context
3. `toast.success("Form submitted! Now proceed to payment")` appears (top-right `sonner` toast)
4. After 500ms delay, `document.getElementById("payment")?.scrollIntoView({ behavior: "smooth" })` scrolls to payment placeholder
5. Form card remains visible above — user can scroll back up to review

## Responsive & Accessibility Requirements

### Mobile Adaptations
- **Touch targets:** All interactive elements (input fields, checkbox, buttons, eye toggle) must have minimum 44×44px touch target area per FORM-09.
- **Form width:** Single column, `max-w-lg` (512px max), full-width on mobile (padding `px-4` on section, card `p-6` reduces to `p-4` on small screens)
- **Card padding:** `p-6 sm:p-8` — smaller padding on mobile
- **Aadhar eye toggle:** Icon-button with `size="icon-xs"` (24px) wrapped in a touch-friendly container ensuring 44×44 minimum tap area
- **Checkbox label:** Clickable label that extends checkbox target area (label `htmlFor` pointing to checkbox `id`)
- **Scrolling:** No horizontal overflow. Single vertical scroll.
- **Keyboard:** Number-only keyboard (`inputMode="numeric"` or `type="tel"`) for Contact Number and Aadhar fields on mobile
- **Zoom:** Form must not disable user zoom (`maximum-scale` not set, font size 16px minimum prevents iOS zoom)

### Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Labels | All inputs have visible `<label>` with `htmlFor` matching input `id`. Never placeholder-only labels (D-11). |
| Required markers | Required fields marked with `<span className="text-destructive">*</span>` inside FieldLabel |
| Error association | `aria-invalid` on input, `aria-describedby` pointing to error message `id`. `FieldError` from shadcn handles this automatically. |
| Error summary | `role="alert"` on error banner — screen reader announces on render |
| Focus management | On submit with errors, `form.setFocus("firstInvalidField")` moves focus to first invalid field |
| Aadhar eye toggle | `aria-label` dynamic: `"Show Aadhar"` / `"Hide Aadhar"`. Button is `type="button"` to prevent form submission. |
| Keyboard navigation | Tab through all fields in logical order. Submit button activatable with Enter. Eye toggle activatable with Enter/Space. |
| Color contrast | Verify `--destructive` (error red) on `--background` (#FEF2F2) passes WCAG AA. Verify `--muted-foreground` on `--card` (#F87171) passes AA for placeholder text. |
| Reduced motion | Guest field slide animation must respect `prefers-reduced-motion`: use Tailwind's `motion-reduce:transition-none motion-reduce:!grid-rows-[1fr] motion-reduce:!opacity-100` pattern. |
| Screen reader announcements | `toast.success()` from sonner announces via `role="status"`. Error banner announces via `role="alert"`. |
| Touch targets | Minimum 44×44 CSS pixels for all interactive elements (FORM-09). |
