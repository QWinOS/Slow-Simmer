<task type="auto">
  <name>Task 2: Create vitest configuration and test setup</name>
  <files>vitest.config.ts, tests/setup.ts</files>
  <action>
    Create two test infrastructure files:

    1. **vitest.config.ts** at project root:
    ```typescript
    import { defineConfig } from "vitest/config"
    import path from "path"

    export default defineConfig({
      test: {
        environment: "jsdom",
        setupFiles: ["./tests/setup.ts"],
        globals: true,
      },
      resolve: {
        alias: {
          "@": path.resolve(__dirname, "."),
        },
      },
    })
    ```
    Reason: jsdom environment is needed for React component tests (rendering DOM). The `@` alias mirrors tsconfig.json's path mapping so tests can use `@/lib/validations` imports. `globals: true` enables `describe`, `it`, `expect` without explicit imports.

    2. **tests/setup.ts** — test framework setup:
    ```typescript
    import "@testing-library/jest-dom/vitest"
    ```
    Per VALIDATION.md Wave 0 requirements — adds jest-dom matchers (toBeInTheDocument, toHaveClass, etc.) to vitest's expect.

    Create the `tests/` directory if it doesn't exist. Use `mkdir -p tests`.

    Do NOT add any vitest scripts to package.json — the executor runs tests via `npx vitest run`.
  </action>
  <verify>
    <automated>npx vitest run --config vitest.config.ts --reporter=verbose 2>&1 | head -20</automated>
  </verify>
