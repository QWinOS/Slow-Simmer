    **lib/drive.ts** — Create typed utility module for Google Drive API interactions.

    Export `DriveFile` interface with fields: `id: string`, `name: string`, `mimeType: string`, `webContentLink?: string`.

    Export `fetchGalleryImages()` async function:
    - Read `NEXT_PUBLIC_DRIVE_FOLDER_ID` and `NEXT_PUBLIC_GOOGLE_API_KEY` from `process.env`
    - If either is missing, return empty array (graceful fallback for dev)
    - Build URL: `https://www.googleapis.com/drive/v3/files` with query params:
      - `q`: `'{folderId}' in parents and mimeType contains 'image/'`
      - `key`: apiKey
      - `fields`: `files(id,name,mimeType,webContentLink)`
      - `pageSize`: `50`
    - Use `new URL()` + `searchParams.set()` pattern (not string concat) for URL building
    - Throw typed error if response not ok
    - Return `data.files || []`
    - Per RESEARCH.md: fetch is the recommended approach for public folders + API key (no googleapis SDK)

    Export `getDriveImageUrl(fileId: string): string`:
    - Return `https://drive.google.com/uc?export=view&id=${fileId}`
    - This direct URL pattern works for files in public folders ("Anyone with the link")

    Do NOT export a `DriveFile` type with optional fields that would create ambiguity. The type must match what Drive API v3 returns for `files.list` with the specified `fields` parameter.

    **lib/video.ts** — Create typed utility module for embed URL generation.

    Export `getYouTubeEmbedUrl(videoId: string, autoplay?: boolean): string`:
    - Use `https://www.youtube-nocookie.com/embed/${videoId}` (privacy-enhanced per D-14/D-15, RESEARCH.md common pitfalls)
    - Params: `rel=0`, `modestbranding=1`, and `autoplay=1` if autoplay flag is true

    Export `getYouTubeThumbnailUrl(videoId: string): string`:
    - Return `https://i.ytimg.com/vi/${videoId}/hqdefault.jpg`

    Export `getInstagramEmbedUrl(postId: string): string`:
    - Return `https://www.instagram.com/p/${postId}/embed`
    - Per RESEARCH.md: direct embed iframe is the simplest approach; does not require oEmbed API

    Per D-13 (images fetched dynamically), D-14/D-15 (video thumbnails with play overlay).

    **.env.local** — Create template file (DO NOT commit real credentials):
