  <name>Task 2.1: Create NavBar client component with smooth-scroll navigation</name>
  <files>components/NavBar.tsx</files>
  <action>
    Create a 'use client' component at `components/NavBar.tsx`.

    **Scroll behavior (D-02):** Use native CSS smooth scroll (set in Plan 1 globals.css). The NavBar links use `<a href="#section-id">` which triggers browser smooth scroll.

    **Props:** None (self-contained).

    **State:**
    - `activeSection: string` — tracks which section is currently in viewport (default "hero")
    - `isScrolled: boolean` — tracks if page has scrolled past hero height (for background transition)
    - `isMobileMenuOpen: boolean` — toggles hamburger menu on mobile

    **Sections to track** (in order, matching section DOM order):
    `["hero", "about", "gallery", "videos", "form"]`

    **IntersectionObserver:**
    - Create in useEffect with `rootMargin: "-64px 0px -40% 0px"` (offset by nav height ~64px, trigger when section is near top 40% of viewport)
    - Observe all section elements by their IDs
    - On intersection change, update `activeSection`
    - Clean up observer on unmount

    **Scroll listener:**
    - Attach scroll event in useEffect
    - If `window.scrollY > 100`, set `isScrolled = true` (adds solid background + subtle shadow to nav)
    - Else `isScrolled = false` (transparent/translucent background)

    **Visual states (per UI-SPEC interaction states):**
    - Default: transparent/translucent background, foreground color text (#450A0A light / #FEF2F2 dark)
    - Scrolled: solid bg-background with shadow (shadow-sm), background transition
    - Active nav link: gold accent underline + font-semibold (color accent-foreground)
    - Hover nav link: gold accent underline (color accent-foreground)
    - Hamburger visible < 640px (md:hidden on desktop nav links)

    **Layout:**
    - Fixed/sticky top (position: sticky, top: 0, z-index: 50)
    - Full width, flex row, justify-between, items-center
    - Left: "Supper Club" brand text (font-heading, text-xl)
    - Right: Desktop nav links + mobile hamburger button
    - Desktop links: horizontal list, gap-x-8, each link is `<a href="#section-id">{label}</a>`
    - Mobile (<640px): hamburger icon (RiMenuFill / RiCloseFill from remixicon) toggles vertical nav list
    - Mobile menu: absolute dropdown below navbar, full width, stacked links
    - Active link: `className={cn(activeSection === id && "text-accent-foreground border-b-2 border-accent")}`
    - Scrolled nav: `className={cn("bg-background/95 backdrop-blur-sm shadow-sm transition-all duration-200")}`

    **Section labels & IDs:**
    ```
    hero → "Hero"
    about → "About"
    gallery → "Gallery"
    videos → "Videos"
    form → "Register"
    ```

    **Accessibility:**
    - Hamburger button has aria-label="Toggle navigation menu"
    - Mobile menu has aria-expanded reflecting open state
    - All links have clear text content

    Per D-02: navigation via smooth scroll links only.
    Per D-03: single scrollable page (no routing — pure hash-based navigation).
    Per UI-SPEC: hamburger at <640px, touch targets minimum 44×44px, sticky nav with background transition.
    Per RESPONSIBILITY MAP: NavBar is Client tier — needs browser APIs (IntersectionObserver, scroll events).

    Do NOT add 'use client' to page.tsx or any Server Component — keep NavBar as isolated client boundary.
    Do NOT use react-scroll or any external smooth-scroll library — native CSS is sufficient and preferred per don't-hand-roll table.
    Do NOT use Link from next/link — hash anchor navigation uses plain <a> tags.
