## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-06-01 | Information Disclosure | NEXT_PUBLIC_GOOGLE_API_KEY reused for Sheets, exposed in client bundle | medium | mitigate | Same control as the Drive gallery: restrict the API key by HTTP referrer in Google Cloud Console; the key only reads public sheet data (spreadsheet shared "Anyone with the link"), no write scope. | T-01-06-02 | Tampering | Malicious/garbage row in the source sheet injects an arbitrary string as a video ID | low | mitigate | fetchVideoLinks never passes a raw URL to an iframe: it runs each cell through detectVideoPlatform + getYouTubeVideoIdFromUrl / getInstagramPostIdFromUrl, which return only a parsed bare id or null; unparseable rows are skipped.
