    **components/GallerySection.tsx** — Client wrapper ('use client'). Replaces the Plan 2 stub.

    This component owns the state for image data and lightbox coordination. It does NOT call fetch directly — that responsibility belongs to GalleryGrid. Instead, it orchestrates the GalleryGrid and GalleryLightbox child components.

    Structure:
    ```typescript
    'use client'

    import { useState } from "react"
    import { GalleryGrid } from "./GalleryGrid"
    import { GalleryLightbox } from "./GalleryLightbox"
    import type { DriveFile } from "@/lib/drive"

    export default function GallerySection() {
      const [images, setImages] = useState<DriveFile[]>([])
      const [selectedIndex, setSelectedIndex] = useState<number | null>(null)

      const openLightbox = (index: number) => setSelectedIndex(index)
      const closeLightbox = () => setSelectedIndex(null)

      return (
        <section id="gallery" className="py-16 sm:py-24 scroll-mt-16">
          <div className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8">
            <h2 className="text-2xl sm:text-3xl font-heading font-bold text-center mb-12 text-foreground">
              Gallery
            </h2>
            <GalleryGrid
              images={images}
              onImagesLoaded={setImages}
              onImageClick={openLightbox}
            />
          </div>
          {selectedIndex !== null && (
            <GalleryLightbox
              images={images}
              currentIndex={selectedIndex}
              onClose={closeLightbox}
              onNavigate={setSelectedIndex}
            />
          )}
        </section>
      )
    }
    ```

    Props passed to GalleryGrid:
    - `images: DriveFile[]` — current image list (empty initially)
    - `onImagesLoaded: (images: DriveFile[]) => void` — callback when Grid fetches data
    - `onImageClick: (index: number) => void` — opens lightbox at clicked image index

    Props passed to GalleryLightbox:
    - `images: DriveFile[]` — full list for navigation
    - `currentIndex: number` — currently displayed image
    - `onClose: () => void` — close handler
    - `onNavigate: (index: number) => void` — change current image

    **components/GalleryGrid.tsx** — Client component ('use client').

    Props:
    ```typescript
    interface GalleryGridProps {
      images: DriveFile[]
      onImagesLoaded: (images: DriveFile[]) => void
      onImageClick: (index: number) => void
    }
    ```

    State:
    - `loading: boolean` — initially true, false after fetch resolves
    - `error: string | null` — null initially, string on failure

    **Data fetching:**
    - useEffect on mount calls `fetchGalleryImages()` from `@/lib/drive`
    - On success: call `onImagesLoaded(data)`, set `loading = false`
    - On error: set `error = error.message`, set `loading = false`
    - Empty array is NOT an error — it's the empty state (no images in folder yet)
    - Per GALL-04: images are fetched dynamically at runtime — no build-time data, no caching of URLs

    **Responsive grid** (per D-10: 2 cols mobile, 3 cols tablet, 4 cols desktop):
    ```html
    <div className="grid grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 gap-4">
    ```

    **Loading state** (when `loading === true` AND `images.length === 0`):
    - Render 6 `<Skeleton>` components in the exact same grid layout
    - Each skeleton: `aspect-[4/3] rounded-xl` (matching image aspect ratio)
    - Use shadcn Skeleton component for the shimmer effect (it has built-in animation)
    - Per UI-SPEC: "6 skeleton cards with shimmer animation, matching grid layout"

    **Error state** (when `error !== null`):
    - Centered flex column with icon, error message, and retry button
    - Remixicon: `RiErrorWarningLine` (size 48, text-muted-foreground)
    - Text: "Unable to load gallery. Please try again later." (per UI-SPEC copy)
    - Retry button: shadcn `<Button variant="outline" onClick={retry}>` where retry re-runs the fetch
    - Per UI-SPEC: "Warning icon + message + retry button"

    **Empty state** (when `loading === false` AND `images.length === 0` AND `error === null`):
    - Centered flex column
    - Remixicon: `RiImageLine` (size 48, text-muted-foreground)
    - Heading: "No photos yet" (per UI-SPEC copy)
    - Body: "Check back soon — new photos will appear here automatically." (per UI-SPEC copy)

    **Loaded state** (when `images.length > 0`):
    - Render each image in a `<Card>` (shadcn Card with no padding/background — just the image container)
    - Use `aspect-[4/3]` for consistent image ratio
    - Image URL: use `getDriveImageUrl(image.id)` from `@/lib/drive`
    - Use native `<img>` element with `loading="lazy"` for lazy loading
    - `alt` text: `image.name || "Supper club event photo"` (dynamic name from Drive, fallback per UI-SPEC a11y)
    - On click: `onImageClick(index)`
    - Hover effect: `hover:scale-105 transition-transform duration-300 cursor-pointer` (per UI-SPEC gallery items interaction states)
    - Rounded corners via Card: `rounded-xl overflow-hidden`

    Per RESPONSIBILITY MAP: Gallery data fetching and grid display are both Client tier — needs browser fetch + state management.
    Per RESEARCH Pitfall 1: API key restrictions (referrer-based) documented in .env.local template.
    Per RESEARCH Pitfall 3: Drive thumbnailLink is short-lived — use `getDriveImageUrl()` (direct `uc?export=view` URL) for reliable display.

    - `<DialogContent className="max-w-5xl w-[95vw] p-0 bg-black/95 border-none">`:

      - The shadcn Dialog already handles overlay + backdrop click
      - Positioned full-viewport centered
    - Inside DialogContent:
      - `<DialogTitle className="sr-only">Photo viewer</DialogTitle>` (screen-reader only per a11y)
      - `<DialogDescription className="sr-only">Viewing photo {currentIndex + 1} of {images.length}</DialogDescription>` (screen-reader only per a11y)

      - **Close button** (top-right, absolute): X button using `<DialogClose asChild>` with a `<Button variant="ghost" size="icon">` containing `RiCloseLine` from remixicon

      - **Navigation arrows** (left/right sides, absolute, vertically centered, hidden on mobile <640px):
        - Left arrow: `<button onClick={() => onNavigate(prevIndex)}>` with `RiArrowLeftSLine` icon
        - Right arrow: `<button onClick={() => onNavigate(nextIndex)}>` with `RiArrowRightSLine` icon
        - Arrows have semi-transparent dark background, rounded-full, hover effect
        - `prevIndex` wraps: `(currentIndex - 1 + images.length) % images.length`
        - `nextIndex` wraps: `(currentIndex + 1) % images.length`
        - Show both arrows always (wrapping navigation — last image wraps to first and vice versa)

      - **Current image**:
        - `<img>` with `src={getDriveImageUrl(images[currentIndex].id)}`
        - `alt={images[currentIndex].name || "Supper club event photo"}`
        - `className="max-h-[85vh] w-full object-contain"` (scale to fit viewport)
        - Initially hidden, animate in on mount (simple CSS fade-in via transition)

      - **Image counter** (bottom-center, absolute):
        - `{currentIndex + 1} / {images.length}` in text-white/80 text-sm

    **Keyboard navigation** (via useEffect):
    - Add `keydown` event listener to the document (or use Dialog's built-in handler)
    - When Dialog is open:
      - ArrowLeft → `onNavigate((currentIndex - 1 + images.length) % images.length)`
      - ArrowRight → `onNavigate((currentIndex + 1) % images.length)`
      - Escape → already handled by shadcn Dialog built-in
    - Clean up listener on unmount
    - NOTE: shadcn Dialog's DialogContent might intercept some keyboard events. The `onOpenChange` prop handles Escape. For arrow keys, attach the listener to the DialogContent ref specifically, or use a portal event listener approach.

    **Touch swipe support** (mobile, per GALL-03):
    - Add `onTouchStart`, `onTouchMove`, `onTouchEnd` handlers to the image container
    - Track `touchStartX` and `touchEndX`
    - If `|touchEndX - touchStartX| > 50px` (minimum swipe threshold):
      - Swipe left (touchEndX < touchStartX) → next image
      - Swipe right (touchEndX > touchStartX) → previous image
    - Per UI-SPEC: "swipe to navigate photos (touch events)"

    **Accessibility** (per UI-SPEC):
    - Focus trap: handled by shadcn Dialog automatically
    - Escape to close: handled by shadcn Dialog automatically
    - Arrow key navigation: via the useEffect handler
    - Screen reader: DialogTitle + DialogDescription provide context
